# EVM 靓号/个性化钱包生成工具 (EVM Vanity Wallet Generator)

一个基于 Node.js **多线程 (Worker Threads)** 实现的高性能 EVM 钱包私钥与地址生成工具。支持通过自定义规则筛选钱包地址（如排除特定数字、仅使用指定数字、指定后缀等）。

---

## 🚀 特性

- **多核加速**：默认启用所有可用的 CPU 核心进行并行计算，极大提升碰撞效率。
- **自定义过滤**：
  - `排除数字`：从地址中剔除指定的数字（例如：排除 4 和 9）。
  - `仅限数字`：限制地址中能出现的十进制数字字符（例如：数字只能是 1, 3, 6, 8，十六进制字母 a-f 依然允许）。
  - `尾部字符集`：规定地址的最后几位（长度等于参数长度）必须仅由指定的字符集组成（例如：指定后缀 `1368`，最后 4 位可以是 `3618`、`8316` 或 `1133` 等任意由 1,3,6,8 组成的组合）。
- **实时监控**：命令行以极低延迟实时刷新生成速率（Keys/second）、总尝试次数、已运行时间及已成功找到的钱包数量。
- **自动存盘**：找到的钱包地址与私钥会即时追加保存至本地文件（默认 `wallets.txt`）。

## 📖 命令行参数说明

| 参数 (长) | 参数 (短) | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `--exclude` | `-e` | `string` | 无 | **排除数字**。指定地址中不允许出现的数字，用逗号分隔或连写（如 `4,9` 或 `49`） |
| `--only` | `-o` | `string` | 无 | **指定组成数字**。地址中允许出现的数字集合（如 `1,3,6,8` 或 `1368`）。未指定的数字将被禁止 |
| `--suffix` | `-s` | `string` | 无 | **尾部字符集限制**。地址最后 N 位（N为参数长度）必须由指定的字符集组成（如 `1368` 允许最后 4 位为 `3618`, `8316` 等） |
| `--count` | `-c` | `string` | `1` | 需要匹配并生成的钱包数量 |
| `--threads` | `-t` | `string` | CPU 核心数 | 并行运行的线程数。默认使用本机的全部可用 CPU 核心 |
| `--output` | `-f` | `string` | `wallets.txt` | 结果输出保存的文件路径 |
| `--help` | `-h` | `boolean`| 无 | 打印帮助信息 |

## 💡 使用示例

### 示例 1：完全满足您的自定义需求
> **需求**：排除数字 4 和 9；出现数字只能有 1, 3, 6, 8；且最后面 4 位由 1, 3, 6, 8 任意组合构成（如 `3618`）。
```bash
node generator.js --exclude 4,9 --only 1,3,6,8 --suffix 1368 --count 1
```

*注意：此组合条件极其严格（概率为一亿四千万分之一），大约需要 1.5 小时的计算时间（视机器配置而定）。*

### 示例 2：快速碰撞（推荐体验）
> **需求**：排除 4 和 9；且最后 4 位是 1, 3, 6, 8 的任意组合（如 `8316`）。不限制其他位置的数字。
```bash
node generator.js --exclude 4,9 --suffix 1368 --count 1
```
*此操作通常在 15 秒内即可完成。*

### 示例 3：纯豹子尾号
> **需求**：尾部 4 位全为 8，且生成 3 个钱包。
```bash
node generator.js --suffix 8888 --count 3
```

## 🔒 安全与建议

1. **完全本地运行**：本工具的代码完全在本地执行，没有任何网络请求，确保您的私钥不会泄露。
2. **私钥保管**：生成的钱包会明文写入 `wallets.txt` 中。在使用完毕后，请务必安全转移私钥并妥善清理此文本文件，以防本地数据被他人窃取。
